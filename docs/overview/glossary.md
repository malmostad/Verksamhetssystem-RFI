# Ordlista

## Systemtermer

### Masterdata
Data som är auktoritativ källa för information inom en datadomän. Exempel: Personaldata i HRutan, Vårddata i Lifecare-Procapita.

### SSO (Single Sign-On)
Enkel inloggning - användare loggar in en gång och får tillgång till flera system.

### API (Application Programming Interface)
Gränssnitt mellan system för datautbyte.

### Integration
Koppling mellan två eller flera system för datautbyte eller funktionalitet.

## Autentisering

### Freja eID
Säker inloggningsmetod med LOA 2/3 (Level of Assurance) som stödjer SSO.

### SITHS
Säker IT i Hälso- och Sjukvården - inloggningsmetod med kort och PIN för vårddata.

### AD (Active Directory)
Microsofts katalogtjänst för användarhantering och autentisering.

## Systemkategorier

### Centrala System
Kritiska system som stödjer verksamhetens kärnfunktioner. Exempel: HRutan, Lifecare-Procapita, Ekot.

### Molntjänster
System som körs i molnet (SaaS). Exempel: CMP, Kuben, Guardtools.

### Applikationer
Specifika applikationer för specifika uppgifter. Exempel: Mobilt e-Siths, SITHS admin.

### Tjänster
Externa tjänster och system. Exempel: NPÖ, Pascal, MCSS.

## Processer

### Kärnprocess
Huvudprocess som stödjer verksamhetens primära uppdrag. Exempel: Larmhantering, Vårdhantering.

### Stödprocess
Process som stödjer kärnprocesser. Exempel: Personalhantering, IT-support.

## Data

### Master System
System som är auktoritativ källa (master) för en datadomän.

### Sekundärt System
System som tar emot data från master system.

### Dataägare
Organisation eller roll som ansvarar för datakvalitet och datapolicy.

## Tekniska termer

### Batch
Dataöverföring som sker i satser (t.ex. dagligen).

### Realtid
Dataöverföring som sker omedelbart.

### HL7
Standard för datautbyte inom hälso- och sjukvården.

### GDPR
General Data Protection Regulation - dataskyddsförordning.

## Kritikalitet

### Kritisk
System som måste vara tillgängliga 24/7 med minimal nedtid (< 1 timme).

### Hög prioritet
System som är viktiga för daglig drift med nedtid < 4 timmar.

### Medel prioritet
Viktiga stödsystem med nedtid < 24 timmar.

### Låg prioritet
Stödsystem med nedtid < 48 timmar.
